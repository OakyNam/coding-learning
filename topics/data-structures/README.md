# Data Structures

Learn how to choose and use Python data structures to store, look up, group, count, deduplicate, sort, and safely update data.

This topic is Python-focused, but the decision-making transfers to other languages: choose the shape of data based on how the program needs to access it.

## Prerequisites

You should already be comfortable with basic Python variables, functions, loops, conditionals, and simple input/output.

## What You Will Learn

- How lists, tuples, dictionaries, and sets differ
- How to choose a structure based on access, order, uniqueness, mutability, and performance needs
- How to use common patterns such as lookup tables, grouping, counting, queues, and sorting
- How standard-library helpers such as `collections.Counter`, `collections.defaultdict`, and `collections.deque` make common tasks easier
- How to avoid common mutation, copying, key, set, and default-argument mistakes
- How to combine these ideas in a small project

## Suggested Learning Order

Use the lessons in order unless you are reviewing a specific idea. The first two lessons build the vocabulary, the third lesson turns containers into useful patterns, the fourth lesson helps you debug real mistakes, and the final project combines everything.

Before starting the project, make sure you can explain when you would choose a `list`, `tuple`, `dict`, or `set`.

## How To Use This Topic

Each lesson includes examples, common mistakes, an exercise, and a worked answer. Try the exercise before reading the worked answer; the moment of getting stuck is part of the learning.

## Completion Goal

By the end, you should be able to choose an appropriate Python data structure and use it in a small program that organizes, searches, counts, groups, and reports data.
